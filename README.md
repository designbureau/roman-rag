# roman-rag — the Cicero Archive

A retrieval-augmented reading interface over **Cicero's complete works**,
presented as a **Latin + English parallel text** and voiced through a **Roman
ensemble** of personas (Cicero, Tiro, Atticus, Caesar, plus a modern Classicist,
an Interpreter, and a Storyteller), all speaking from the same retrieval layer.

It is a standalone fork of the Bleek-Lloyd archive architecture, re-pointed at a
new corpus. See **`docs/cicero-archive-plan.md`** for the full plan, the locked
design decisions, and the validated Perseus parallel-text alignment findings.

> **Status: vertical slice in progress.** The monorepo skeleton is in place with
> the data model adapted for Cicero, and the **Letters to Atticus ingestion is
> live** — `pnpm scrape --source=letters-att` pulls Perseus Latin (Purser) +
> English (Shuckburgh) and aligns them at whole-letter granularity (414/446 =
> 92.8% parallel). The persona rewrite, embedding/chat deploy, analytics, and
> frontend re-theme are next per the plan.

## Stack

- **Frontend**: React Router v7 (framework mode, Vite) · shadcn/ui · Tailwind v4 · Vercel AI SDK `useChat` — deployed to Vercel
- **Backend**: Supabase (Postgres + pgvector) · Edge Functions for `chat`, `search`, `speak`, `transcribe`
- **Generation**: Anthropic Claude (Sonnet) via the Edge Function (per-persona temperature)
- **Embeddings**: OpenAI `text-embedding-3-small` (1536 dim)
- **CLI**: Node 20 + TypeScript for local scrape + embed (uses `@supabase/supabase-js` REST client for upserts)

## Corpus & sources

Primary source: **Perseus `canonical-latinLit`, author `phi0474` (Cicero)** — a
`git clone` into `data/.cache/perseus/`. Latin and English editions are aligned
by canonical citation ref (e.g. `Att. 1.5`). See plan §4.

## Running (once the corpus + Supabase project exist)

1. Provision a Supabase project (pgvector). Set Edge Function secrets `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, `ELEVENLABS_API_KEY`.
2. Copy `.env.example` → `.env` and populate `SUPABASE_URL`, `SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY`.
3. `pnpm install`
4. `pnpm scrape` — ingest Cicero sources → `data/stories.json`
5. `pnpm embed` — populate `chunks` with embeddings
6. `pnpm topics` — build the static analytics/glossary JSON
7. Frontend env: set `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY` in `web/.env.local`
8. `pnpm dev` — frontend at http://localhost:5173

Edge Functions: `pnpm functions:serve` (local), `pnpm functions:deploy:chat` / `:search` (or via Supabase MCP `deploy_edge_function`).
